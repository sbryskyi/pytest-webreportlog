# pytest-webreportlog

## Project Purpose

A web-based viewer/dashboard for pytest-reportlog output.

## Dependencies Analysis

### Core Dependencies:
- **pytest>=8.4.2** - Standard Python testing framework
- **pytest-reportlog** (local editable from `../pytest-reportlog`) - Pytest plugin that generates test execution logs in JSON Lines format
- **ruff>=0.14.3** - Python linter/formatter

### Architecture:
- `pytest-reportlog` generates structured test reports in JSONL format
- This project (pytest-webreportlog) aims to transform that JSONL output into a web-based interface
- Makes test results easier to analyze compared to raw JSON data

### JSONL Report Structure:
The `pytest-reportlog` output includes:
- Session start/finish events
- Collection reports
- Test reports with timing data (setup, call, teardown phases)
- Outcomes, locations, and keywords for each test
- Precise timestamps and durations

## Web Service

A FastAPI-based web service for viewing and managing pytest test session reports.

### Technology Stack

- **Backend**: FastAPI + SQLModel
- **Database**: SQLite
- **Frontend**: Jinja2 templates + Tailwind CSS + HTMX
- **Server**: Uvicorn

### Features

1. **Upload JSONL Reports**: Upload pytest-reportlog JSONL files via web UI
2. **Session List**: View all test sessions with summary statistics
3. **Detailed View**: Drill down into individual test results by phase (setup/call/teardown)
4. **REST API**: Programmatic access to session data
5. **Phase Isolation**: View stdout/logs/errors for each test phase separately

### Quick Start

1. **Install dependencies**:
   ```bash
   uv sync
   ```

2. **Start the server**:
   ```bash
   ./server.sh start
   ```

3. **Open browser**:
   ```
   http://localhost:8000
   ```

4. **Upload a report**:
   - Click "Upload JSONL Report" button
   - Select a `.jsonl` file generated by pytest-reportlog
   - View the session details

5. **Stop the server**:
   ```bash
   ./server.sh stop
   ```

### Server Management

The `server.sh` script provides simple server management:
- `./server.sh start` - Start server in background
- `./server.sh stop` - Stop server
- `./server.sh restart` - Restart server
- `./server.sh status` - Check server status

Server logs are written to `server.log`.

### API Endpoints

- `GET /` - Web UI home page (list of sessions)
- `GET /sessions/{id}` - Session detail page
- `POST /upload` - Upload JSONL file
- `GET /api/sessions` - List all sessions (JSON)
- `GET /api/sessions/{id}` - Get session details (JSON)

### Database

SQLite database is stored in `data/sessions.db` and contains:
- **Session** table: Test session metadata and summary statistics
- **TestReport** table: Individual test reports for each phase

## Testing

The project includes two comprehensive test suites organized in separate directories:

### Report Format Tests (`tests/reports/`)
Validates pytest report generation:
- JSONL report generation and structure (15 tests)
- JUnit XML report format (14 tests)
- HTML report format (15 tests)
- Exception handling across all test phases
- Log/stdout capture and isolation by phase

```bash
# Run report format tests
uv run pytest tests/reports/ -v
```

### Web Server Tests (`tests/web/`)
Validates the web service functionality (20 tests):

**Part 1: Unit Tests** (`test_web_server.py`) - JSONL upload, parsing, and display
- Upload and parse JSONL reports
- Session statistics accuracy
- HTML generation and formatting
- Separate entries for setup/teardown errors
- Traceback formatting

**Part 2: Comparison Tests** (`test_html_comparison.py`) - Ensures compatibility with pytest-html
- Compares session statistics with pytest output
- Verifies setup/teardown error handling matches pytest-html
- Validates result counts and labeling

```bash
# Run web server tests
uv run pytest tests/web/ -v

# Run all tests
uv run pytest tests/
```

See [tests/web/README.md](tests/web/README.md) for detailed web server test documentation.
