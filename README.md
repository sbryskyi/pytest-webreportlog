# pytest-webreportlog

A web-based viewer/dashboard for pytest-reportlog output.

## Overview

This project provides a web interface for viewing and analyzing pytest test results generated by [pytest-reportlog](https://github.com/pytest-dev/pytest-reportlog). It transforms JSONL test reports into an interactive dashboard with features like test history tracking, real-time streaming, and detailed phase-by-phase analysis.

## Features

### Core Features
- **Upload JSONL Reports**: Upload pytest-reportlog JSONL files via web UI or API
- **Session Management**: View, filter, and delete test sessions with summary statistics
- **Detailed Test View**: Drill down into individual test results by phase (setup/call/teardown)
- **Phase Isolation**: View stdout/logs/errors for each test phase separately with ANSI color rendering
- **Dark Mode**: Toggle between light and dark themes with persistent preference

### History & Analytics
- **Test History Overview**: Aggregated statistics across all sessions with pass rate tracking
- **Individual Test History**: Track test stability over time with session-by-session breakdown
- **Sortable Tables**: Sort by test name, runs, pass rate, duration, or last run

### Real-time Features
- **Streaming API**: Real-time test results via Server-Sent Events (SSE)
- **Live Updates**: Automatic UI refresh for in-progress sessions
- **Event Posting**: Post test events directly to running sessions

## Technology Stack

- **Backend**: FastAPI + SQLModel + SQLite
- **Frontend**: Jinja2 templates + Tailwind CSS + HTMX
- **Server**: Uvicorn
- **CLI**: Typer

## Quick Start

### Installation

```bash
uv sync
```

### Start the Server

```bash
# Development mode with auto-reload
pytest-webreportlog dev

# Or run in foreground
pytest-webreportlog serve

# Or run in background (daemon mode)
pytest-webreportlog start
```

### Open Browser

Navigate to http://localhost:8000

### Upload a Report

1. Click "Upload JSONL Report" button
2. Select a `.jsonl` file generated by pytest-reportlog
3. View the session details

### Generate a Report

```bash
# Run pytest with reportlog
pytest --report-log=report.jsonl

# Then upload report.jsonl via the web UI
```

## CLI Commands

The `pytest-webreportlog` CLI provides server management:

```bash
# Development server with auto-reload
pytest-webreportlog dev

# Start server in foreground
pytest-webreportlog serve [--host HOST] [--port PORT] [--reload] [--workers N]

# Start server in background (daemon mode)
pytest-webreportlog start [--host HOST] [--port PORT] [--pid-file PATH] [--log-file PATH]

# Stop background server
pytest-webreportlog stop [--pid-file PATH]

# Restart background server
pytest-webreportlog restart [--host HOST] [--port PORT]

# Check server status
pytest-webreportlog status [--pid-file PATH]

# Get help
pytest-webreportlog --help
pytest-webreportlog serve --help
```

### Examples

```bash
# Development with auto-reload (default: localhost:8000)
pytest-webreportlog dev

# Production with multiple workers
pytest-webreportlog serve --host 0.0.0.0 --port 8080 --workers 4

# Background daemon with custom files
pytest-webreportlog start --port 9000 --pid-file myserver.pid --log-file myserver.log
```

## Database Configuration

The server uses SQLite for data persistence. By default, it creates/uses `data/sessions.db`.

**Custom database**: Use the `DATABASE_URL` environment variable:

```bash
DATABASE_URL=sqlite:///data/myproject.db pytest-webreportlog serve
```

**Multiple instances**: Run separate instances for different projects:

```bash
# Project A on port 8000
DATABASE_URL=sqlite:///data/project_a.db pytest-webreportlog serve --port 8000

# Project B on port 8001
DATABASE_URL=sqlite:///data/project_b.db pytest-webreportlog serve --port 8001
```

## API Endpoints

### Web UI
| Endpoint | Description |
|----------|-------------|
| `GET /` | Home page (list of sessions) |
| `GET /sessions/{id}` | Session detail page |
| `GET /history` | Test history overview |
| `GET /history/{nodeid}` | Individual test history |

### REST API
| Endpoint | Description |
|----------|-------------|
| `GET /api/sessions` | List all sessions (JSON) |
| `GET /api/sessions/{id}` | Get session details (JSON) |
| `DELETE /api/sessions/{id}` | Delete session and test reports |
| `POST /upload` | Upload JSONL file |

### Streaming API
| Endpoint | Description |
|----------|-------------|
| `POST /api/stream/event` | Post individual test events |
| `GET /api/stream/{session_id}` | Subscribe to real-time updates (SSE) |

### Streaming Example

```bash
# Terminal 1: Subscribe to session updates
curl -N http://localhost:8000/api/stream/1

# Terminal 2: Post test events
SESSION_UUID=$(uuidgen)

# Start session
curl -X POST http://localhost:8000/api/stream/event \
  -H "X-Session-ID: $SESSION_UUID" \
  -H "Content-Type: text/plain" \
  -d '{"pytest_version": "8.4.2", "$report_type": "SessionStart"}'

# Post test result
curl -X POST http://localhost:8000/api/stream/event \
  -H "X-Session-ID: $SESSION_UUID" \
  -H "Content-Type: text/plain" \
  -d '{"nodeid": "test_example.py::test_pass", "outcome": "passed", "when": "call", "duration": 0.001, "$report_type": "TestReport"}'

# Finish session
curl -X POST http://localhost:8000/api/stream/event \
  -H "X-Session-ID: $SESSION_UUID" \
  -H "Content-Type: text/plain" \
  -d '{"exitstatus": 0, "$report_type": "SessionFinish"}'
```

## JSONL Report Structure

The `pytest-reportlog` output includes:
- Session start/finish events with pytest version and exit status
- Collection reports
- Test reports with timing data (setup, call, teardown phases)
- Outcomes, locations, and keywords for each test
- Precise timestamps and durations
- Captured stdout/stderr and log output per phase
